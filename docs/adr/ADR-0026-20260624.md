# ADR-0026: Use JWT with Redis Blacklist Instead of Opaque Tokens

**Status:** Accepted
**Date:** 2026-06-24
**Author:** P-r-e-m-i-u-m
**Reviewers:** Engineering Team

---

## Context

Our initial authentication system used opaque tokens stored in PostgreSQL. Every authenticated request required a database lookup to validate the token, adding 15-40ms latency per request.

## Decision

We will use JWT tokens (1h expiry) with a Redis blacklist for invalidation. On logout, tokens are added to the blacklist with TTL matching remaining token lifetime.

## Alternatives Considered

| Option | Pros | Cons | Decision |
|-|-|-|-|
| Chosen approach | Best fit for requirements | Some tradeoffs | Accepted |
| Alternative A | Simpler | Missing key features | Rejected |
| Alternative B | More features | Too complex / costly | Rejected |

## Consequences

### Positive
Auth overhead reduced from 15-40ms to 1-3ms. Immediate token invalidation on logout without waiting for expiry.

### Negative
Redis blacklist must be replicated across all instances.
